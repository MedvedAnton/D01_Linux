## Part 3. Настройка сети ОС
- 1. Задаем новое название машины: `sudo hostnamectl set-hostname user-1`
	или установить так sudo 

- 2. Задал свой регион командой `timedatectl set-timezone Asia/Yekaterinburg`

- 3. Cетевые интерфейсы - `ip link show` или `ip l`

	![](screens/ip_link_show.png)

	lo (loopback) — виртуальный сетевой интерфейс, присутствует	в любой
	операционной системе с поддержкой сетевого стека.
	Позволяет приложениям на локальной машине обращаться к сетевым сервисам,
	запущенным на этой же машине, без использования физического сетевого
	оборудования. 
	
- 4. Получаем ip `ip addr show` или `ip a`
	Вывод будет в графе enp0s3, после слова inet.
	
	![вывод команды](screens/ip_addr_show.png)

	DHCP (Dynamic Host Configuration Protocol) - протокол, который
	автоматически назначает устройствам в сети сетевые настройки.
	Компьютер-клиент на этапе конфигурации сетевого устройства обращается
	к серверу DHCP и получает от него нужные параметры.

- 5. Внешний ip-адрес шлюза - `curl ifconfig.co`

	![](screens/ip1.png)

	Внутренний ip адресс шлюза - `ip route`, в 1ой строке после слов default via
	
	![](screens/ip2.png)

- 6. Задаем статичные сетевые настройки. 
	Откроем файл конфигурации netplan в редакторе vim:
	`sudo vim /etc/netplan/00-installer-config.yaml`
	`i` что бы ввести текст.
	Изменим параметр dhcp4 на значение false.
	Далее прописываем: 
	---
	`addresses: [10.0.0.9/24]`  # укажем статический IP-адрес\
	`gateway4: 10.0.2.2`  # зададим внутренний IP-адрес \
	`nameservers:` \
	  `[8.8.8.8, 1.1.1.1]` # укажем публичные DNS серверы.
	---
	esc что бы закончить редактирование
	`:wq` выйти и сохранить
	`sudo netplan apply` принять изменения
	
	![изменяем](screens/netplan.png)

- 7. Перезагружаем машину командой `reboot`
	командой `cat /etc/netplan/00-installer-config.yaml` проверяем настройки
	
	![проверяем](screens/netplan_check.png)
	
	Пингуем 
	
	![пингуем](screens/ping.png)

## Part 8. Установка и базовая настройка сервиса SSHD
- sshd (Secure Shell Daemon) — серверная служба в Linux, обеспечивает
	безопасный удалённый доступ к системе по протоколу SSH.
	
- Устанавливаем службу: `sudo apt install openssh-server`

- Включить автостарт при запуске: `sudo systemctl enable ssh`\
	Выключить автостарт: `sudo systemctl disable ssh`\
	Проверяем статус: `sudo systemctl status ssh`
	
- Меняем порт: 
	Открываем конфигурационный файл: `sudo nano /etc/ssh/sshd_config`\
	Убираем `#` перед Port, если есть. Ставить значение `2022`
	
	![Port 2022](screens/ssh_config.png)
	
	Для сохранения перезапустим службу: `sudo systemctl restart ssh`
	
	![порт изменился](screens/port2022.png)
	
- Отображение наличия процесса sshd командами: \
	`ps -C sshd` покажет все процессы с именем sshd.\
	`ps aux | grep sshd` выведет подробную информацию по процессам, связанным с sshd.\
	`ps -ef | grep sshd` аналогично предыдущему, но в другом формате вывода.
	
	![ssh](screens/ssh.png)

- `ps` -- утилита для просмотра списка процессов.
- Основные и часто используемые флаги:\
	`-A`, `-e`	Показать все процессы в системе\
	`-a`	все, кроме лидеров сессий и без управляющего терминала\
	`-d`	все, кроме лидеров сессий\
	`-N`	Инвертировать выбор (всё, кроме указанных)\
	`-C <имя>`	только с указанным именем команды\
	`-G <ID/имя>`  принадлежащие указанной группе\
	`-p <PID>`	процесс с указанным PID (можно перечислить несколько через запятую)\
	`--ppid <PPID>`	 дочерние процессы указанного родительского процесса\
	`-s <SID>`	принадлежащие указанной сессии\
	`-t <tty>`	связанные с указанным терминалом\
	`-u <пользователь>`	указанного пользователя\
	`-U <пользователь>`	указанного пользователя по реальному UID (кто запустил процесс)

	Лидер сессии — процесс, от которого происходят все остальные процессы в рамках
	одной сессии (например, все команды, запущенные в одном терминале).\
	PID (Process IDentifier) — уникальный числовой идентификатор процесса.\
	UID (User IDentifier) —  уникальный числовой идентификатор пользователя.

- Вывод netstat -tan

	![](screens/netstat.png) 

- `netstat` показывает статистику приема и отправки пакетов и информацию об ошибках приема и отправки.

	Флаги:\
	`-t` — отображать только TCP-соединения.\
	`-a` — показать все активные соединения и порты, которые находятся в состоянии
	прослушивания (LISTEN).\
	`-n` — выводить адреса и порты в числовом формате, без попытки преобразовать
	их в имена сервисов или хостов.

	При вызове команды в терминал выводится таблица, столбцы которой имеют следующие значения:
	1) Proto -- сетевой протокол (tcp, udp)
	
	2) Recv-Q -- Количество байтов, помещённых в буфер приёма TCP/IP, но не переданных приложению.
	
	3) Send-Q -- Количество байтов, помещённых в буфер отправки TCP/IP, но не отправленных,
	или отправленных, но не подтверждённых.
	
	4) Local Address — Локальный адрес сервера. Формат: IP:порт.
	
	5) Foreign Address — Удалённый адрес и порт, с которым установлено соединение.
	
	6) State — 	Состояние соединения (например, LISTEN — ожидание новых соединений,
	ESTABLISHED — соединение установлено).
	
- 0.0.0.0 в колонке Local Address означает, что сервис слушает на всех доступных сетевых
	интерфейсах сервера. К этому порту можно подключиться с любого IP-адреса, если нет
	других ограничений (например, фаервола). Если указан конкретный IP (например, 127.0.0.1),
	сервис будет доступен только с этого интерфейса.

## Part 14. Работа с системными журналами
- `less имя_файла` — просмотрщик текстовых файлов (и потоков данных) с возможностью прокрутки в обе стороны,
	поиском и навигацией. 
	
- /var/log/dmesg — файл журнала, содержит сообщения, генерируемые ядром системы.
	В нём хранится информация о событиях, связанных с состоянием аппаратных устройств и драйверами,
	инициализироваными во время загрузки системы и её работы. Включает сообщения ошибках,
	предупреждениях и другой информации, связанной с работой ядра.
	
	![less /var/log/dmesg](screens/dmesg.png)
	
- /var/log/syslog — основной системный журнал, содержит сообщения обо всех событиях,
	происходящих в системе с момента её запуска.
	
	![less /var/log/syslog](screens/syslog.png)
	
- /var/log/auth.log — журнал авторизации, содержит информацию о попытках входа пользователей в систему,
	а также о механизмах аутентификации, которые были использованы. В этом файле фиксируются как успешные,
	так и неудачные попытки входа, что делает его важным инструментом для мониторинга безопасности системы.
	
	![less /var/log/auth.log](screens/auth.png)
	
- Последний успешный вход:\
	Пользователь: dante\
	Время: 20:28:10\
	Метод входа:(login:session) -- через локальный, вместо login может быть например ssh
	
	![less /var/log/auth.log](screens/less_log.png)
	
	Либо командой `last | head -10` -- показывает последние 10 логов\
	Надпись tty1 — значит вход выполнен через локальную консоль\
	Другие варинаты входа:\
	pts/* — удалённый вход (SSH, telnet)\
	:0, :1 — графический вход (X11/Wayland)
	
	![last](screens/last.png)
	
- Рестарт службы ssh: `sudo systemctl restart ssh`
- Проверка `less /var/log/auth.log | tail -10`

	![проверяем последние 10 логов](screens/restart_ssh.png)
